Case Study

Mac Markdown Workspace

A Bear-inspired, local-first Markdown notes workspace — tag sidebar, document list, and a distraction-free, structure-colored editor — built on a bespoke design system and shipped from one React codebase to macOS desktop, the browser, and iOS.

Live Demo

Problem

Build a local-first notes product that feels like Bear on the Mac — fast, quiet, typography-first — while reaching the browser and mobile from the same code. One design system, one component tree, three platforms, no rewrites.

Design System

The system's central rule: structure is colored, prose is not. Heading hashes, list bullets, checkboxes, quote bars, and link brackets render in the active theme's accent — in both the CodeMirror editor and the rendered preview — while the words stay neutral. Four palettes (Teal, Forest, Gold, Crimson), each with a derived dark variant, live as CSS custom properties; switching themes restyles the entire app, editor included, with zero re-renders.

Implementation Highlights

  • Notes library with live-derived titles, previews, and #hashtags — content is the single source of truth; tag filtering and instant search across everything
  • One platform-agnostic React app behind a typed host-capability API: notes persist as real markdown files on macOS, IndexedDB on web, and Capacitor Filesystem on iOS
  • Custom CodeMirror 6 highlight theme keyed to design tokens — one rule colors every structural mark in the accent
  • Aa popover with nine curated editor faces, a text-size scale, palette swatches, and light/dark/system — all persisted per platform
  • Standalone themed HTML and PDF export with WebKit-specific delivery (share sheet on touch, print-tab on Safari) that survives iOS transient-activation rules
  • Bear-style mobile kit: bottom tool strip plus a markdown helper bar that rides above the on-screen keyboard and inserts at the cursor
  • Autosave with flush-on-switch; import flows create notes rather than overwrite them

Verification Rigor

Every phase shipped against a running app, not just unit tests: driven Chromium and real Safari via WebDriver, Playwright WebKit with iPad emulation, and the packaged Electron app over CDP — down to asserting that exported files land on disk. The hardest bug was invisible to all of it: a clipped dropdown that programmatic clicks happily "pressed" while no human could see it. The fix — and the lesson, hit-testing UI with elementFromPoint — is now part of the project's verification playbook.

Current Status

  • Live on the web; macOS desktop and iOS builds from the same tree
  • 11-phase design overhaul tracked as Gherkin user stories on GitHub — 12 of 13 shipped and closed
  • 56 unit tests plus Playwright e2e against the packaged desktop app
